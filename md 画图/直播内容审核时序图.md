sequenceDiagram
    participant View as 主播端/观众端 (View)
    participant Ctrl as 风控控制器 (Controller)
    participant Serv as 业务服务层 (Service)
    participant FFmpeg as 媒体引擎 (FFmpeg)
    participant Python as ASR微服务 (Faster-Whisper)
    participant DFA as DFA多线程池 (WorkerPool)
    participant DB as 数据库 (MySQL)

    %% 语音审核流程
    rect rgb(245, 245, 245)
        Note over View, DB: 语音流旁路智能审核机制 (定时/流式触发)
        Serv ->> FFmpeg: 旁路请求提取最新3秒音频切片
        FFmpeg -->> Serv: 返回WebM/Opus二进制碎片
        Serv ->> Python: HTTP POST /transcribe 提交音频
        
        Note over Python: 执行动态降噪与音量归一化<br/>采用"3s切片+1s重叠"防止语意截断
        Python ->> Python: 注入违规词 Prompt 引导模型推理
        Python -->> Serv: 返回高精度转录文本 (Text)
    end

    %% 文本与语音文本统一过滤
    rect rgb(240, 248, 255)
        Note over View, DB: 统一文本风控过滤流程 (弹幕/转录文本)
        View ->> Ctrl: 发送实时交互文本 (如弹幕)
        Ctrl ->> Serv: 发起敏感词检测请求
        
        Serv ->> DFA: 调度空闲 Worker 线程分配检测任务
        Note over DFA: 执行全角转半角、剔除零宽字符等预处理
        Note over DFA: 分级状态树匹配 (1级->4级), 1级短路返回
        DFA -->> Serv: 返回命中结果 {hit, level, matchedWords}
    end

    %% 违规阻断机制
    alt 检测未命中或级别为 4 (Log)
        Serv ->> DB: 异步写入弹幕/放行日志
        Serv -->> Ctrl: 返回检测通过
        Ctrl -->> View: 正常广播与展示
    else 检测命中 1~2 级 (Reject)
        Serv ->> DB: 记录至 sensitive_hit_log 并增加 hit_count
        Serv ->> DB: 更新 user_black_white_list 加入黑名单
        
        Note over Serv, FFmpeg: 若为主播语音违规，触发强制阻断
        Serv ->> FFmpeg: 调用 closeLiveStream 强制掐断流传输
        
        Serv -->> Ctrl: 返回拦截异常
        Ctrl -->> View: WebSocket 立即推送 "违规封禁/下播" 警告弹窗
    end
